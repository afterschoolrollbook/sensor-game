📂 STEP 1 · 센서 연결 섹션 열기

  1. SGP 실행 → 스텝6 (게임 진행) 으로 이동
  2. 상단 "센서 연결" 섹션 클릭해서 펼치기
  3. 상태 표시가 // 미연결 로 보이면 정상

🔌 USB 유선 방식

  1. ESP32를 USB 케이블로 PC에 연결
  2. "포트 연결" 버튼 클릭
  3. Chrome 팝업에서 포트 선택 (예: COM5)
  4. 상태가 ✅ 연결됨 으로 바뀌면 완료
⚠️
Chrome 또는 Edge 브라우저에서만 작동합니다. Firefox·Safari는 지원하지 않습니다.

📡 WiFi 방식

  1. ESP32에 WiFi 코드 업로드
  2. 시리얼 모니터 열기 → IP 주소 확인
    IP: 192.168.0.105  ← 이 숫자 복사
  3. SGP IP 입력창에 복사한 IP 입력
  4. "연결" 버튼 클릭 → ✅ 연결됨 확인
ℹ️
PC와 ESP32가 반드시 같은 WiFi 공유기에 연결되어 있어야 합니다.

🔵 동글 (ESP-NOW) 방식

  1. 수신용 ESP32 (동글) → 수신 코드 업로드
  2. 시리얼 모니터에서 MAC 주소 확인 후 복사
    MAC: 24:6F:28:AB:CD:EF
  3. 송신용 ESP32 코드의 receiverMac에 입력 후 업로드
  4. 수신용 ESP32를 USB로 PC에 연결
  5. SGP에서"동글 포트 연결" 클릭 → 포트 선택
  6. ✅ 동글 연결됨 확인
  7. 송신용 ESP32에 보조배터리 연결 → 전원 ON
✅
공유기 없이도 동작! 야외 행사에 최적입니다.

ℹ️

센서 추가 방법

  1. 원하는 동작의 "+ 센서 추가" 버튼 클릭
  2. 핀 번호 입력 (실제 ESP32에 꽂은 핀 번호)
  3. 센서 종류 드롭다운에서 선택
  4. 필요 없으면 ✕ 버튼으로 삭제

⚙️ 코드 자동 생성

  1. 센서 핀 설정 완료 후
  2. "📋 Arduino / ESP32 코드 자동 생성" 버튼 클릭
  3. "📋 복사" 버튼 클릭
  4. Arduino IDE에 붙여넣기 → 업로드 (→)

생성 코드 예시 — WiFi 방식 (타임어택)

#include <WiFi.h>
#include <WebSocketsServer.h>

const char* ssid     = "공유기이름";    // ← 수정
const char* password = "WiFi비밀번호";  // ← 수정

WebSocketsServer ws(81);

const int PIN_START_2  = 2;   // ▶ 시작 — 적외선센서
const int PIN_STOP_14  = 14;  // ⏹ 정지 — 적외선센서
const int DEBOUNCE = 300;

void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) delay(500);
  Serial.println("IP: " + WiFi.localIP().toString()); // SGP에 입력!
  ws.begin();
  pinMode(PIN_START_2, INPUT_PULLUP);
  pinMode(PIN_STOP_14, INPUT_PULLUP);
}

void loop() {
  ws.loop();
  if (!digitalRead(PIN_START_2)) { ws.broadcastTXT("START"); delay(DEBOUNCE); }
  if (!digitalRead(PIN_STOP_14)) { ws.broadcastTXT("STOP");  delay(DEBOUNCE); }
}

📶 신호 방식

방식설명값 범위예시 센서핀 제약
디지털 ON/OFF 2가지 0 또는 1 터치, 적외선, 버튼 아무 디지털 핀
아날로그 연속 수치값 0~4095 압력, 조도, 마이크 D32~D39만 가능
I2C SDA+SCL 2선 통신 주소 기반 MPU-6050, BME280 SDA=D21, SCL=D22 고정
SPI 4~6선 고속 통신 패킷 기반 RFID RC522 SCK=D18, MISO=D19, MOSI=D23 고정
UART TX/RX 직렬 통신 문자열 GPS, 바코드 TX=D16, RX=D17
💡
I2C는 예외적으로 같은 D21/D22 핀에 여러 센서를 동시 연결할 수 있습니다. 각 센서가 I2C 주소로 구분되기 때문입니다.

void setup() { Serial.begin(115200); pinMode(TRIG, OUTPUT); pinMode(ECHO, INPUT); }
long getDistance() {
  digitalWrite(TRIG, LOW); delayMicroseconds(2);
  digitalWrite(TRIG, HIGH); delayMicroseconds(10); digitalWrite(TRIG, LOW);
  return pulseIn(ECHO, HIGH) / 58; // cm
}
void loop() {
  if (getDistance() < 10) { send("START"); delay(300); } // 10cm 이내
}

🪪 식별·인식 센서

RFID 리더 RC522 7핀 SPI
카드/태그 인식 · 3.3V 전용!
▾
3.3V→3.3V / GND→GND / SCK→D18 / MOSI→D23 / MISO→D19 / SS→D5 / RST→D22
⚠️
반드시 3.3V! 5V 연결 시 모듈 손상됩니다. 라이브러리: MFRC522
#include <SPI.h>
#include <MFRC522.h>
MFRC522 rfid(5, 22); // SS=D5, RST=D22
void setup() { SPI.begin(); rfid.PCD_Init(); }
void loop() {
  if (rfid.PICC_IsNewCardPresent() && rfid.PICC_ReadCardSerial()) {
    send("FUSE");
    rfid.PICC_HaltA();
    delay(300);
  }
}

제스처 센서 APDS-9960 4핀 I2C
손 제스처로 시작/정지
▾

#include <SparkFun_APDS9960.h>
SparkFun_APDS9960 apds;
void setup() { apds.init(); apds.enableGestureSensor(); }
void loop() {
  if (apds.isGestureAvailable()) {
    switch(apds.readGesture()) {
      case DIR_UP:    send("START"); break;
      case DIR_DOWN:  send("STOP");  break;
      case DIR_LEFT:  send("FUSE");  break;
      case DIR_RIGHT: send("LAP");   break;
    }
    delay(300);

❓ 자주 묻는 질문

포트 선택 창이 안 떠요▾
Chrome 또는 Edge 브라우저를 사용하세요. Firefox·Safari는 Web Serial API를 지원하지 않습니다.
WiFi 연결이 안 돼요▾
① ESP32와 PC가 같은 WiFi 공유기에 연결되어 있는지 확인
② 시리얼 모니터에서 IP 주소 재확인
③ ESP32 코드의 WiFi 이름/비밀번호가 정확히 입력됐는지 확인
센서를 눌러도 반응이 없어요▾
① 수신 로그 확인 → 신호가 찍히면 SGP 함수 문제
② 신호가 없으면 배선 또는 연결 문제
③ 수동 테스트 버튼으로 SGP 자체 동작 먼저 확인
동글 연결은 됐는데 신호가 안 와요▾
송신용 ESP32 코드의 receiverMac을 수신용 MAC 주소로 정확히 교체했는지 확인하세요. MAC 주소는 시리얼 모니터에서 확인합니다.
핀 번호는 마음대로 정해도 되나요?▾
디지털 센서는 아무 디지털 핀이나 가능합니다. 단 각 핀에 센서 하나씩만 연결하세요. 가이드 문서의 핀 번호는 예시이며, 실제로는 본인이 꽂은 핀 번호를 코드에 입력하면 됩니다.
같은 동작에 센서를 여러 개 추가해도 되나요?▾
네! 예를 들어 퓨즈에 적외선 4개를 추가하면 어느 한 곳이라도 감지되면 퓨즈가 실행됩니다. 단 각 센서는 서로 다른 핀에 연결해야 합니다.
